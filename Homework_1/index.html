Git comands

Main

git config --global user.name назначаєм імя користувача (для всіх проектів)

git config --global user.email назначаєм пошту користувача (для всіх проектів)

git init - Створення репозиторію в папці, де виконується команда.

git status - для визначення, які файли в якому стані знаходяться.

git add - додає вміст робочої папки, для наступного комміту.

git commit (-m) - додає коментар, які саме були зроблені зміни.

git log - історія коммітів.

git checkout - Команда для створення гілок або перемикання між ними.

git branch - Переглянути локальні гілки.

git push - Дозволяє помістити зміни у головну гілку на GitHub.

git pull - Для об'єднання змін, з віддаленого репозиторія, в локальний робочий каталог.

git merge - Об'єднує гілки, в активну (основну) гілку.

git clone - Використовується для клонування репозиторію.

Advanced

git revert - Відкат коміта шляхом створення нового, зміни у якому зворотні тому комміту, який треба скасувати.

git reset - Cкидання індексу та робочого каталогу до останнього стану комміту.

git rebase - Для коммітів в іншій гілці.

git cherry-pick - Бере зміни, що вносяться одним коммітом, та намагається повторно застосувати їх у вигляді нового комміту в поточній гілці.

git stash - Допомагає у збереженні змін на тимчасовій основі, ці зміни не потраплять у коміт одразу.
